# 프론트엔드 작업 가이드

이 문서는 프론트엔드 작업을 맡은 준영 님, 승헌 님를 위한 설명 문서입니다.

프론트엔드는 사용자가 직접 보는 화면을 만드는 파트입니다. 이번 프로젝트에서는 단순히 화면을 예쁘게 만드는 것보다, 사용자가 자연스럽게 학식 메뉴를 확인하고 영양 요약까지 볼 수 있도록 흐름을 만드는 것이 중요합니다.

---

## 1. 프론트엔드 앱의 역할

프론트엔드는 백엔드에서 받은 학식 메뉴와 AI 영양분석 결과를 사용자에게 보여줍니다.

주요 흐름은 다음과 같습니다.

```text
오늘의 학식 확인
→ 식당별 메뉴 확인
→ 메뉴 상세 확인
→ 먹을 메뉴 선택
→ 선택한 메뉴들의 영양소 합산 확인
→ 1일 권장량 대비 비교 확인
```

---

## 2. 주요 화면

이번 프로젝트의 프론트엔드는 다음 화면을 중심으로 구성합니다.

```text
1. 오늘의 학식 화면
2. 메뉴 상세 화면
3. 선택한 메뉴 요약 화면
```

화면 수를 너무 늘리기보다, 이 세 화면이 자연스럽게 연결되도록 만드는 것이 중요합니다.

---

## 3. 오늘의 학식 화면

오늘의 학식 화면은 사용자가 앱을 열었을 때 가장 먼저 보는 화면입니다.

이 화면에서는 여러 식당의 메뉴를 식당별로 묶어서 보여줍니다.

사용자가 할 수 있어야 하는 행동은 다음과 같습니다.

```text
- 오늘 날짜의 메뉴를 확인한다.
- 날짜를 변경한다.
- 식당별 메뉴를 확인한다.
- 메뉴를 눌러 상세 화면으로 이동한다.
- 먹을 메뉴를 체크한다.
```

이 화면은 준영 님이 주로 담당합니다.

---

## 4. 메뉴 상세 화면

메뉴 상세 화면은 특정 학식 메뉴의 자세한 정보를 보여주는 화면입니다.

표시할 정보는 다음과 같습니다.

```text
- 식당명
- 날짜
- 식사 구분
- 메뉴 제목
- 원본 메뉴 구성
- 예상 열량
- 예상 탄수화물
- 예상 단백질
- 예상 지방
- 예상 나트륨
- AI 식단 코멘트
```

이 화면은 준영 님과 승헌 님의 작업이 모두 연결될 수 있습니다.

준영 님은 메뉴 목록에서 상세 화면으로 이동하는 흐름을 담당하고, 승헌 님은 영양정보 표시 방식과 요약 화면과의 연결을 함께 맞춥니다.

---

## 5. 선택한 메뉴 요약 화면

선택한 메뉴 요약 화면은 사용자가 체크한 메뉴들의 영양정보를 합산해서 보여주는 화면입니다.

표시할 정보는 다음과 같습니다.

```text
- 선택한 메뉴 목록
- 총 예상 열량
- 총 탄수화물
- 총 단백질
- 총 지방
- 총 나트륨
- 1일 권장량 대비 비율
- 간단한 종합 코멘트
```

이 화면은 승헌 님이 주로 담당합니다.

이 화면이 있어야 앱이 단순한 메뉴 조회 앱이 아니라 식단관리 앱처럼 보입니다.

---

## 6. 준영 님과 승헌 님의 협업 포인트

준영 님과 승헌 님은 서로 다른 화면을 맡지만, 작업은 강하게 연결되어 있습니다.

반드시 함께 맞춰야 할 부분은 다음과 같습니다.

```text
- 메뉴 카드 디자인
- 메뉴 선택 체크박스 동작
- 선택한 메뉴 상태를 공유하는 방식
- 메뉴 상세 화면으로 이동하는 방식
- 요약 화면으로 이동하는 버튼 위치
- 영양정보 표시 방식
```

프론트엔드 작업의 목표는 각자 화면을 따로 만드는 것이 아닙니다.

사용자가 아래 흐름을 자연스럽게 따라갈 수 있어야 합니다.

```text
학식 메뉴 확인
→ 먹을 메뉴 선택
→ 메뉴 상세 확인
→ 선택한 메뉴 요약 확인
```

---

## 7. 샘플 데이터로 먼저 작업하기

처음부터 실제 API 연결을 하려고 하면 작업이 어려워질 수 있습니다.

따라서 먼저 샘플 데이터로 화면을 만듭니다.

샘플 데이터도 실제 API 응답과 비슷한 구조로 만드는 것이 좋습니다.

이렇게 하면 나중에 백엔드 API가 연결될 때 수정할 부분이 줄어듭니다.

---

## 8. 작업 완료 기준

프론트엔드 작업은 다음 기준으로 완료 여부를 확인합니다.

```text
- 앱이 실행된다.
- 맡은 화면이 정상적으로 표시된다.
- 버튼이나 체크박스가 의도대로 동작한다.
- 화면 이동이 자연스럽다.
- 오류 메시지가 없다.
```
