# 승헌 님 작업 가이드: 메뉴 선택 및 영양 요약 화면

## 1. 이번 작업의 목적

이번 작업은 사용자가 선택한 학식 메뉴들의 영양정보를 합산해서 보여주는 화면을 만드는 것입니다.

사용자는 오늘 먹었거나 먹을 메뉴를 여러 개 선택할 수 있고, 앱은 선택한 메뉴들의 예상 영양정보를 합산해 1일 권장량과 비교해 보여줍니다.

---

## 2. 이 작업을 통해 배우는 것

이번 작업을 하면서 다음을 익히는 것이 목표입니다.

```text
- 선택된 데이터를 화면에 표시하는 방법
- 여러 데이터의 숫자 값을 합산하는 방법
- 비율 정보를 UI로 표현하는 방법
- 요약 화면을 구성하는 방법
- 다른 화면의 선택 상태와 연결하는 방법
- AI 코딩 에이전트에게 단계적으로 UI 작업을 요청하는 방법
- Git 브랜치를 만들어 안전하게 작업하는 방법
```

---

## 3. 작업 전에 읽을 문서

작업 전에 아래 문서를 읽습니다.

```text
1. docs/01_VIBE_CODING_GUIDE.md
2. docs/02_GIT_BRANCH_GUIDE.md
3. docs/03_FRONTEND_GUIDE.md
```

---

## 4. 작업 브랜치 만들기

main 브랜치에서 직접 작업하지 않습니다.

아래 명령어로 작업 브랜치를 만듭니다.

```bash
git switch main
git pull origin main
git switch -c feature/nutrition-summary-screen
```

이미 브랜치를 만들어둔 상태라면 다음 명령어로 이동합니다.

```bash
git switch feature/nutrition-summary-screen
```

---

## 5. 담당 기능

승헌 님이 담당하는 기능은 다음입니다.

```text
- 메뉴 선택 기능
- 선택한 메뉴 목록 표시
- 선택한 메뉴들의 영양소 합산 화면
- 1일 권장량 대비 비교 화면
- 선택 메뉴 요약 화면
```

---

## 6. 작업 순서

아래 순서대로 작업합니다.

```text
1. 샘플 선택 메뉴 데이터를 준비한다.
2. 선택한 메뉴 목록을 보여주는 화면을 만든다.
3. 총 열량, 탄수화물, 단백질, 지방, 나트륨을 표시한다.
4. 1일 권장량 대비 비율을 표시한다.
5. 간단한 종합 코멘트 영역을 만든다.
6. 준영 님의 메뉴 선택 화면과 연결한다.
```

처음에는 실제 API 연결 없이 샘플 데이터로 구현합니다.

---

## 7. AI에게 요청할 때의 방향

AI에게는 한 번에 모든 기능을 맡기지 말고, 작은 단위로 요청합니다.

좋은 작업 순서는 다음과 같습니다.

```text
1. 선택한 메뉴 목록 UI 만들기
2. 총 영양정보 카드 만들기
3. 권장량 대비 비율 표시 만들기
4. 종합 코멘트 영역 만들기
5. 메뉴 선택 화면과 연결하기
```

AI에게 요청한 뒤에는 반드시 앱을 실행해서 화면이 잘 나오는지 확인합니다.

---

## 8. 준영 님과 맞춰야 할 부분

승헌 님의 요약 화면은 준영 님의 메뉴 목록 화면에서 선택한 메뉴를 기반으로 동작합니다.

따라서 아래 내용은 준영 님과 함께 맞춥니다.

```text
- 메뉴 선택 체크박스 동작 방식
- 선택한 메뉴 데이터 구조
- 요약 화면으로 이동하는 버튼 위치
- 메뉴 카드와 요약 화면의 디자인 톤
- 선택 해제 동작
```

프론트엔드는 두 사람이 따로 만든 화면을 붙이는 것이 아니라, 하나의 흐름으로 이어져야 합니다.

---

## 9. 완료 기준

아래 조건을 만족하면 이번 작업이 완료된 것으로 봅니다.

```text
- 선택한 메뉴 목록이 보인다.
- 총 영양정보가 보인다.
- 1일 권장량 대비 비율이 보인다.
- 메뉴 선택 화면에서 요약 화면으로 이동할 수 있다.
- 앱 실행 중 오류가 없다.
```

## 10. 커밋하고 PR 올리기

작업이 끝나면 커밋하고 GitHub에 올립니다.

```bash
git status
git add .
git commit -m "feat: add nutrition summary screen"
git push origin feature/nutrition-summary-screen
```

그다음 GitHub에서 Pull Request를 만들고 팀원 A에게 확인을 요청합니다.
